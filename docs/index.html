Документация TGBOT SOLAR API

Это документация для плагина Telegram Bot API для Solar2D, созданного @xqss_DEVELOPER. Плагин позволяет создавать Telegram-ботов с поддержкой текстовых сообщений, медиа, inline-кнопок, callback-запросов, polling и вебхуков.

Установка

Скопируйте файл telegram_bot.lua в корень проекта и инициализируйте бота с токеном от @BotFather.

local telegram = require("telegram_bot")

-- тут инициализируем бота
telegram.init({
    token = "YOUR_BOT_TOKEN",
    botName = "@YourBotName"
})

Параметры:

  • token: Токен бота (обязательный).
  • botName: Имя бота, например, "@MyBot" (необязательный).

Константы

Плагин поддерживает константы, которые заменяются на данные пользователя или чата из последнего события. Используйте их в текстах сообщений, подписях медиа или ответах на callback-запросы.

Доступные константы:

  • $first_name$: Имя пользователя.
  • $last_name$: Фамилия пользователя.
  • $age$: Возраст пользователя (если указан).
  • $last_message_text$: Текст последнего сообщения.
  • $username$: Имя пользователя (@username).
  • $chat_id$: ID чата.

Пример использования:

telegram.sendMessage(nil, "привет, $first_name$! твой id: $chat_id$", nil, function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("сообщение ушло!")
    end
end)

Отправка Сообщений

Метод sendMessage отправляет текстовое сообщение. Если chatId не указан, используется последний активный чат.

telegram.sendMessage(nil, "привет, $first_name$! как дела?", { parseMode = "Markdown" }, function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("сообщение ушло!")
    end
end)

Параметры:

  • chatId: ID чата (необязательный, по умолчанию последний чат).
  • text: Текст сообщения (до 4096 символов, поддерживает константы).
  • options: Настройки (необязательный):
    • parseMode: "Markdown" или "HTML".
    • disableWebPagePreview: true/false для отключения предпросмотра ссылок.
  • callback: Функция обратного вызова (result, error).

Отправка Медиа

Метод sendMedia отправляет фото, видео, аудио или файлы. Если chatId не указан, используется последний активный чат.

telegram.sendMedia(nil, "photo", "https://example.com/image.jpg", {
    caption = "фото для $first_name$!",
    parseMode = "Markdown"
}, function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("медиа отправлено!")
    end
end)

Параметры:

  • chatId: ID чата (необязательный, по умолчанию последний чат).
  • type: Тип медиа ("photo", "video", "audio", "document").
  • media: URL или file_id медиа.
  • options: Настройки (необязательный):
    • caption: Подпись (до 1024 символов, поддерживает константы).
    • parseMode: "Markdown" или "HTML".
  • callback: Функция обратного вызова (result, error).

Inline-Кнопки

Метод sendInlineKeyboard отправляет сообщение с интерактивными кнопками. Если chatId не указан, используется последний активный чат.

telegram.sendInlineKeyboard(nil, "выбери, $first_name$:", {
    {{text = "кнопка 1", callback_data = "btn1"}},
    {{text = "кнопка 2", callback_data = "btn2"}}
}, function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("кнопки отправлены!")
    end
end)

Параметры:

  • chatId: ID чата (необязательный, по умолчанию последний чат).
  • text: Текст сообщения (поддерживает константы).
  • keyboard: Массив кнопок, где каждый элемент — массив объектов:
    • text: Текст кнопки.
    • callback_data: Данные, отправляемые при нажатии (до 64 байт).
  • callback: Функция обратного вызова (result, error).

Callback-Запросы

Что такое Callback-Запросы?

Callback-запросы — это события, которые Telegram отправляет боту при нажатии пользователем inline-кнопки. Они содержат данные из callback_data и позволяют реагировать на действия (например, показать уведомление).

Метод answerCallbackQuery подтверждает обработку запроса и может отправить уведомление.

Параметры answerCallbackQuery

  • callbackQueryId: ID callback-запроса (обязательный).
  • text: Текст уведомления (необязательный, до 200 символов, поддерживает константы).
  • showAlert: true/false, показывать всплывающее окно (false по умолчанию).
  • callback: Функция обратного вызова (result, error).

Примеры Использования

1. Простой ответ:

telegram.answerCallbackQuery(callbackQueryId, "ты нажал кнопку, $first_name$!", false, function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("ответ отправлен!")
    end
end)

2. Всплывающее уведомление:

telegram.answerCallbackQuery(callbackQueryId, "действие подтверждено, $username$!", true, function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("уведомление отправлено!")
    end
end)

3. Подтверждение без текста:

telegram.answerCallbackQuery(callbackQueryId, nil, false, function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("запрос подтвержден!")
    end
end)

4. Обработка в polling:

local function eventHandler(eventType, data)
    if eventType == "callback_query" then
        if data.data == "btn1" then
            telegram.answerCallbackQuery(data.callbackQueryId, "кнопка 1, $first_name$!", true)
        else
            telegram.answerCallbackQuery(data.callbackQueryId, "другое, $username$", false)
        end
    end
end

-- запускаем polling
telegram.startPolling(eventHandler)

Polling

Метод startPolling запускает обработку сообщений и событий в реальном времени.

local function eventHandler(eventType, data)
    if eventType == "command" and data.command == "start" then
        telegram.sendMessage(data.chatId, "добро пожаловать, $first_name$!", nil)
    elseif eventType == "callback_query" then
        telegram.answerCallbackQuery(data.callbackQueryId, "ты кликнул: $last_message_text$")
    end
end

-- запускаем polling
telegram.startPolling(eventHandler)

Параметры:

  • eventHandler: Функция, обрабатывающая события:
    • eventType: Тип события ("command", "message", "callback_query").
    • data: Данные события (зависят от типа).

Вебхуки

Методы setWebhook и deleteWebhook настраивают вебхуки для продакшена.

telegram.setWebhook("https://your-server.com/webhook", function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("вебхук настроен!")
    end
end)

-- убираем вебхук
telegram.deleteWebhook(function(result, error)
    if error then
        print("ошибка: " .. error)
    else
        print("вебхук удален!")
    end
end)

Параметры:

  • url: URL вебхука (для setWebhook).
  • callback: Функция обратного вызова (result, error).
